  🔍 현재 프로젝트 분석

  당신의 grpc-test의 강점:
  1. ✅ Dynamic gRPC 호출 (컴파일 없이 런타임 실행)
  2. ✅ Load Profile 지원 (SINGLE, CONSTANT_THROUGHPUT, RAMP_UP, BURST)
  3. ✅ Virtual Users & RPS 제어
  4. ✅ ValueGenerator 시스템 (FIXED, RANDOM, UUID, SEQUENCE)
  5. ✅ 기본 통계 (success/fail count)

  grpcui와의 차이점 & 당신 프로그램을 써야 하는 이유:
  - grpcui: 단순 단일 요청 테스트 도구 (개발용)
  - 당신의 도구: 부하 테스트 + 성능 측정 전문 도구 (운영/QA용)

  🎯 추가하면 좋을 핵심 기능들

  1. 실시간 통계 대시보드 ⭐⭐⭐

  - 현재 RPS (Real-time)
  - Success/Fail Rate
  - 응답 시간: P50, P95, P99, Max
  - Active Connections
  - Total Requests
  - Error Rate Graph (시간대별)

  2. Metadata 지원 (grpcui처럼)

  gRPC 헤더 설정:
  - Authorization: Bearer token
  - Custom-Header: value
  - Trace-Id, Request-Id 등

  3. 다중 서비스/메서드 동시 테스트

  Command 포트: 100 RPS
  Query 포트: 500 RPS
  Streaming 포트: 50 RPS
  MarketData 포트: 1000 RPS
  → 4개 포트 동시 부하 테스트

  4. InfluxDB + Grafana 연동

  메트릭 수집:
  - latency_ms{service="OrderService", method="CreateOrder"}
  - request_count{status="success"}
  - error_rate
  → Grafana 실시간 대시보드

  5. Response Body 검증

  Assert 기능:
  - response.status == "OK"
  - response.orderId != null
  - response.price > 0
